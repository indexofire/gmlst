# Internal Documentation

Internal docs are split into two lifecycle zones:

- `stable/` — active documents used for ongoing engineering decisions
- `archive/` — historical snapshots kept for reference

## Structure

- `stable/README.md` — rules for active internal docs
- `archive/README.md` — archived index and retention notes

### Stable starter docs

- `stable/ongoing_architecture_report.md`
- `stable/archive_migration_report.md`

## Current state

At the moment, all existing internal docs are archived snapshots.
New investigations/designs should start under `stable/`.

## Notes

- `AGENTS.md` is maintained only at repository root.
- User-facing docs remain under `docs/`.
